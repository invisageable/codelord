# CL-CL02 — 21 April 2026

## Fixed — Magic zoom now covers overlays

Holding `Cmd+E` while the file picker, a popup menu, or the unsaved-changes
dialog was open used to zoom the main app body only — the overlay stayed
pinned at 1×. Now all overlays (popups, file picker, dialogs, toasts)
zoom in lockstep with the main body.

### Why it was broken

Overlays render via `egui::Area` at the `Context` level, outside the
`scope_builder` wrap in `fn ui`. `ctx.set_transform_layer` only transforms
the specific layer it's called on — it doesn't cascade to sibling layers.
The magic zoom transform was applied exclusively to `magic_zoom_layer`,
so anything painted into a different layer stayed at 1×.

### How it's fixed

After all overlays have painted in `fn logic`, enumerate every visible
layer via `ctx.memory(|m| m.layer_ids())` and apply the current transform
to each, skipping `magic_zoom_layer` (already transformed). On idle
frames, `TSTransform::IDENTITY` is pushed instead, which egui treats as
"remove entry" — clears any stale transforms from a just-finished zoom.

Works for dynamic popup ids (`popup_{entity:?}`) without hardcoding.

### Supersedes

The [CL-CL01 note](./CL-CL01-20-04-2026.md) stating "overlays stay at 1×
by design" no longer applies — the design is now "everything zooms,
including overlays."

- `apps/coder/codelord-coder/src/coder.rs` — three helpers on `impl Coder`
  (`magic_zoom_layer_id`, `magic_zoom_transform`, `propagate_magic_zoom`);
  refactored `fn ui` to use them; one-line call from `fn logic`.
