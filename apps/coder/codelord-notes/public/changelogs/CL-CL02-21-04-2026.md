# CL-CL02 — 21 April 2026

## Fixed — Magic zoom now covers overlays

Hold `Cmd+E` with the file picker, a popup menu, or the unsaved-changes
dialog open and they zoom in lockstep with the main body. Previously
they stayed pinned at 1×.

## Renamed — `file_picker` → `filescope`

The Telescope-inspired fuzzy finder is now called `filescope`. "File
picker" implied a native OS file dialog (a different thing, still
available via the voice command "open file" and `dialog.rs`). Supersedes
the [CL-CL01 note](./CL-CL01-20-04-2026.md) referring to the feature by
the old name.

## Fixed — Play button respects the playlist

Adding a track to the playlist and hitting play now starts that track.
Previously only double-clicking a row started playback; the play button
itself did nothing until something was already loaded. Behavior now
matches macOS Music.

## Changed — No hardcoded demo track

Removed the copyrighted `freeze-corleone-desiigner-a-colors-show.mp3`
bootstrap. An empty playlist with a press-play does nothing (the user
has to add a track, as in any real player). The asset file itself
remains — will be deleted in a follow-up.

## Refactored — Internal architecture

Two cleanups with no behavior change, worth recording for contributors:

- **AudioDispatcher resource** — `codelord_audio`'s `LazyLock<Sender>`
  global is now wrapped in a zero-sized ECS resource. Callers fetch it
  from the world instead of importing `codelord_audio::music_play` etc.
  directly; the audio dependency becomes visible in ECS schedules.
- **Feature plugin pattern (Phase 1)** — each ECS feature in
  `codelord-core` gains `install(world)` + `register_systems(schedule)`
  free functions. `Coder::new()` calls them per feature instead of
  holding every feature's setup inline. Applied to `theme`, `page`,
  `audio`, `magic_zoom`, `filescope` as a proof of concept; remaining
  ~25 features will follow the same pattern in a later pass.
