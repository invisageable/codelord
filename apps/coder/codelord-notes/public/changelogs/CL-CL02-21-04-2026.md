# CL-CL02 тАФ 21 April 2026

## Fixed тАФ Magic zoom now covers overlays

Hold `Cmd+E` with the file picker, a popup menu, or the unsaved-changes
dialog open and they zoom in lockstep with the main body. Previously
they stayed pinned at 1├Ч.

## Renamed тАФ `file_picker` тЖТ `filescope`

The Telescope-inspired fuzzy finder is now called `filescope`. "File
picker" implied a native OS file dialog (a different thing, still
available via the voice command "open file" and `dialog.rs`). Supersedes
the [CL-CL01 note](./CL-CL01-20-04-2026.md) referring to the feature by
the old name.

## Fixed тАФ Play button respects the playlist

Adding a track to the playlist and hitting play now starts that track.
Previously only double-clicking a row started playback; the play button
itself did nothing until something was already loaded. Behavior now
matches macOS Music.

## Changed тАФ No hardcoded demo track

Removed the copyrighted `freeze-corleone-desiigner-a-colors-show.mp3`
bootstrap. An empty playlist with a press-play does nothing (the user
has to add a track, as in any real player). The asset file itself
remains тАФ will be deleted in a follow-up.

## Refactored тАФ Internal architecture

Two cleanups with no behavior change, worth recording for contributors:

- **AudioDispatcher resource** тАФ `codelord_audio`'s `LazyLock<Sender>`
  global is now wrapped in a zero-sized ECS resource. Callers fetch it
  from the world instead of importing `codelord_audio::music_play` etc.
  directly; the audio dependency becomes visible in ECS schedules.
- **Feature plugin pattern (Phases 1тАУ3)** тАФ every ECS feature in
  `codelord-core` now exposes `install(world)` + `register_systems(schedule)`
  free functions. `Coder::new()` calls them per feature instead of
  holding every feature's setup inline. Applied to all ~30 features
  (theme, page, audio, magic_zoom, filescope, animation, loading, color,
  navigation, statusbar, panel, previews, search, popup, tabbar,
  drag_and_drop, keyboard, xmb, about, settings, git, instruction, toast,
  terminal, voice, codeshow, symbol, text_editor, playground).
- **Preview file-type predicates moved into their modules** тАФ each
  `previews/{sqlite,pdf,xls,font,svg}.rs` now exposes
  `pub fn accepts(path) -> bool`. The `is_*_file` helpers in `coder.rs`
  are gone; adding a new preview type is fully self-contained in its
  module (state + `install` + `accepts` + `register_systems`).
- **Phase 4 тАФ the rest of the sprawl** тАФ extractor registration moved
  to `codelord_language::install_{symbol,token}_extractors`; context
  menus moved to their owning features (`settings::spawn_popup`,
  `navigation::spawn_context_popup`, `tabbar::spawn_context_popup`,
  `previews::sqlite::spawn_export_popup`); initial window chrome and
  statusbar icons moved to `titlebar::spawn_default` and
  `statusbar::spawn_default_icons`; non-Send SVG cache moved to
  `components::renderers::svg::install_non_send`; runtime + voice
  visualizer wrap into `runtime::install` and
  `codelord_voice::install_visualizer`.
- `coder.rs` dropped from **3074 тЖТ 2433 LoC** (тИТ641). Zero inline
  `codelord_core::X` references remain in function bodies.
- **Stripped all wasm32 conditional compilation** тАФ codelord is a native
  desktop app, never built for wasm. Removed ~30 `#[cfg(not(target_arch
  = "wasm32"))]` attributes and deleted ~6 WASM-only placeholder items
  (duplicate `XlsData` struct + impl, stub `open`/`select_sheet` bodies,
  empty `register_systems` for wasm). Moved `gilrs`, `raw-window-handle`,
  `swisskit` from `[target.'cfg(not(target_arch = "wasm32"))'.dependencies]`
  to plain `[dependencies]` in codelord-coder + codelord-core
  Cargo.toml.
- **Playground setup consolidated** тАФ `playground::install` now spawns
  the output/time metric entities and inserts `StagebarResource` with
  the compiler-stages preset; `playground::spawn_default_tab` spawns
  the zo hello-world first-run tab. Fixes a silent double-insert of
  `PlaygroundMetrics` (default then overwritten with entity ids).
- **Compilation event handler moved to playground** тАФ
  `playground::apply_compilation_event(world, event)` replaces the
  99-line `Coder::handle_compilation_event`. The function only mutates
  playground resources; it doesn't belong in the app shell. Added
  `codelord-protocol` as a direct dep of `codelord-core` (pure DTOs,
  zero internal cycle risk).
- **Session state moved to codelord-core** тАФ `SessionState` DTO +
  `from_world` (extract) + `apply_to_world` (restore) now live in
  `codelord_core::session`. `Coder::restore_session` shrinks from
  ~160 LoC of ECS writes to a ~20-line storage-reader shell. The
  app-shell module (`codelord-coder::session`) keeps only
  `SESSION_KEY` + `clear_session(storage)` тАФ the eframe-storage
  transport bits. Added `serde` as a direct dep of `codelord-core`
  for the DTO derives.
