# CL-CL03 — 22 April 2026

## Fixed — Terminal keyboard input

Typing into the bottom-panel terminal now actually types. The text
editor view was proactively spawning `ClearFocusRequest` on every frame
where a click landed outside its own rect — including clicks on the
terminal. The terminal's own `focus.set(terminal_entity)` would land
first, then next frame `clear_focus_system` would clear it, so the
pre-consume block in `terminal/view.rs` would see `has_focus == false`
and drop every keystroke. Removed the proactive clear: sibling
focus-grabbing widgets (terminal, filescope, …) set focus to themselves
on click, which implicitly drops the editor's focus, so the explicit
clear was never doing anything useful and was racing.

## Fixed — Playground had no stage buttons

`playground::install` was never being called in `Coder::new`. The
feature-plugin refactor on 21 April missed it when wiring the install
list, so `StagebarResource`, `PlaygroundMetrics`, `PlaygroundOutput`,
`PlaygroundFeedback`, and `PlaygroundWebviewState` were never inserted
and `stagebar::show` early-returned with the resource missing. Added
the install call; stage buttons render again.

## Fixed — Magic zoom bled past the rounded corners

Zooming in made content visible in the four rounded-corner "bites" of
the app window, because the zoomed content layer is transformed but
the rounded cutouts of the border are just painted outlines that don't
clip. Two pieces now fix it:

- `magic_zoom::propagate` now also skips the `app_border` layer, so
  the border (and anything painted on it) stays anchored to the OS
  window at 1×.
- When the zoom is engaged, `magic_zoom::mask_corners` paints four
  `window_fill`-filled concave shapes into those corner bites on the
  border layer. At identity zoom the masks are skipped so the corners
  stay transparent and the OS background shows through.

## Changed — Voice visualizer status has one source of truth

`VisualizerStatus` used to exist twice: once in `codelord-voice` and
once in `codelord-core`, with a 14-line match in `coder.rs` mapping
every variant one-to-one. Moved the enum to
`codelord_protocol::voice::VisualizerStatus` (both sibling crates
already depend on protocol) and re-exported from both. `coder.rs`
loses the useless mapping.

## Changed — `CompileRequest` carries the protocol `Stage` end-to-end

The ECS `CompileRequest` used `stage: u8` (magic numbers 0–4), then
`sdk.compile` did a `u8 → "Tokens" / "Tree" / ...` string match that
duplicated `codelord_protocol::compilation::Stage` variant names. Now
both the ECS component and the SDK function take `Stage` directly, and
the SDK body serialises a `protocol::CompileRequest` literal — no
u8-to-string ritual.

## Changed — `icon::components::Language` name collision left alone

Considered renaming to `IconLanguage` to stop reading as a duplicate of
`codelord_core::language::Language`. Reverted: the `icon::components`
enum follows a strict `Variant(Variant)` convention (`Arrow(Arrow)`,
`Folder(Folder)`, `Zoom(Zoom)`) and prefixing with `Icon` would break
the local pattern. Rust's module system disambiguates the two at import
sites; no change needed.

## Changed — `Selected` marker back where it's used

A second `Selected` unit-struct in `ui::component::selected` was dead
code with zero importers; the live one lives in
`navigation::components` alongside `FileEntry` / `Expanded`. Deleted
the dead copy and its re-export. Consumers (`navigation::systems`,
`explorer.rs`) now import directly from `navigation::components`.

## Changed — `page::SlideDirection` → `page::TransitionDirection`

Two unrelated `SlideDirection` enums used to collide (`codeshow` =
`Next/Previous/First/Last` for presentation nav; `page` = `Left/Right`
for page-to-page slide animation). Renamed the page one so grep-ing
`SlideDirection` only points to the presenter feature.

## Refactored — Gilrs remote control is now an ECS feature

`Coder` no longer owns a `gilrs::Gilrs` handle or a 55-line poll block
in `logic`. Moved to `codelord_core::remote`:

- `RemoteResource(Option<gilrs::Gilrs>)` — non-send resource wrapper.
- `poll_remote(world)` — exclusive system that drains events, maps
  NORWII-N76-style buttons to `SlideDirection`, and spawns
  `NavigateSlide` when a presentation is loaded.
- `install` / `register_systems` so `Coder::new` just adds two lines.
- `gilrs` dep moved from `codelord-coder/Cargo.toml` into
  `codelord-core/Cargo.toml`.

## Refactored — Window animations moved into core

`ShakeAnimation` already lived in `codelord-core::animation::resources`
but the 30-line shake/center tick logic was inline in `coder.rs`'s
`fn logic`. Added:

- `ShakeAnimation::tick(now) -> WindowAnimationStep` (damped sine,
  signed-seed offsets).
- `CenterWindowAnimation::new / ::tick` (OutExpo easing).
- A new `WindowAnimationStep { x, y, finished }` type.

`coder.rs` now just feeds `ctx.input(|i| i.time)` to `tick`, sends the
returned `(x, y)` via `ctx.send_viewport_cmd(OuterPosition)`, and
clears the option when `finished`.

## Refactored — `mod.rs` purged

Renamed every `foo/mod.rs` to sibling `foo.rs` across all codelord
crates (20 files in `codelord-core`, `codelord-components`,

## Refactored — Comment pass on `coder.rs`

Applied Stack Overflow's "[Best Practices for Writing Code Comments][1]"
to drop ~80 non-doc comments that restated the next line
(`// Calculate center position`, `// Despawn the request entity`,
`// Set loading state`, keyboard-shortcut labels that just repeated
`egui::Key::X`, section headers like `// Setup Systems Schedule`). 11
inline comments remain, each explaining *why* — SVG non-Send
constraint, magic-zoom layering, "Must run BEFORE voice sync" ordering
invariant, `show_animated_inside` hidden-body gotcha, corner-mask
conditional, SQLite/PDF open "stays here because of block_on/background
thread", etc. File is down from 1448 → 1336 LoC with no behavior
change.

[1]: https://stackoverflow.blog/2021/12/23/best-practices-for-writing-code-comments/

## Added — `curl | sh` installer with SHA-256 check

`tasks/install.sh` — single POSIX `sh` script a user can pipe from
`https://codelord.sh/install.sh` (or the raw GitHub URL). It:

1. Detects macOS `arm64` / `x86_64`, rejects anything else up front.
2. Hits `api.github.com/repos/compilords/codelord/releases/{latest
   | tags/$CODELORD_VERSION}`, parses the JSON with
   `grep`/`sed` (no `jq` dep).
3. Downloads `codelord-macos-<arch>-<version>.tar.gz` and its
   `.sha256` sidecar, verifies locally with `shasum -a 256`.
4. Untars into `/Applications` (or `$CODELORD_PREFIX` — `sudo`-promoted
   only if the target isn't writable).
5. Strips `com.apple.quarantine` so Gatekeeper stops blocking launch
   on the unsigned/un-notarized bundle.

Env overrides: `CODELORD_VERSION` (default `latest`),
`CODELORD_PREFIX` (default `/Applications`).

Install guide at `codelord-notes/public/guidelines/01-install.md`.

## Added — `release.sh` emits and uploads a SHA-256 sidecar

After `tar czvf`, `shasum -a 256` now writes
`<ARTIFACT>.sha256` next to the tarball; `gh release create` uploads
both. `install.sh` fails hard if the sidecar is missing or mismatches.

## Changed — `Info.plist` patch uses `plutil`, not `sed`

`bundle-codelord.sh` was inserting `NSMicrophoneUsageDescription` via a
`sed 's|</dict>|...|'` replacing the first `</dict>` it found. That
works today but silently breaks the day anyone adds a nested dict to
`Info.plist` (`CFBundleURLTypes`, `CFBundleDocumentTypes`, etc.) —
the first `</dict>` stops being the root. Swapped to `plutil -insert`
which targets the root dict unambiguously. Still idempotent via the
existing `grep -q NSMicrophoneUsageDescription` guard.
