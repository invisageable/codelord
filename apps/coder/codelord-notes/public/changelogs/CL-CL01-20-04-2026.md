# CL-CL01 — 20 April 2026

## Added — Magic Zoom (Screen-Studio-style camera)

Hold `Cmd+E`: the editor smoothly zooms ~2× toward the cursor. Release:
eases back to 1×. Designed for demos, screen recordings, and anyone who
wants a quick visual magnifier.

### Behavior

- **Hotkey**: `Cmd+E` held (not toggle). Matches the codelord held-key
  modality (same pattern as `Cmd+Shift+Space` for voice push-to-talk).
- **Scope**: the whole app body — titlebar, search panel, central content,
  statusbar, and music player — all zoom together.
- **Overlays stay at 1×** by design: popups, file picker, toasts remain
  readable during zoom. Matches the Screen Studio convention of exempting
  UI chrome from camera motion.

### Design

Two smoothing models, one per use case:

| Channel | Model | Why |
|---|---|---|
| Zoom factor | `eazy::Easing::InOutCubic`, 0.30 s duration | Discrete event (engage/disengage). Clear start and stop. |
| Camera (x, y) | Exponential smoothing, τ = 0.08 s | Continuous cursor tracking. Frame-rate-independent, no "restart" lag. |

Duration-based easing on a moving target is a classic trap — each retarget
resets the curve to its slow-start region and the follower never catches
up. Exponential smoothing gives bounded steady-state lag (`v · τ`), which
reads as natural camera weight.

### Architecture

New feature module under the standard codelord ECS pattern:

```
codelord-core/src/magic_zoom/
  mod.rs        # re-exports
  messages.rs   # MagicZoomCommand { engage: bool }
  resources.rs  # MagicZoomState + Eased + Smoothed primitives
  systems.rs    # update_magic_zoom_system
```

Input layer (`codelord-coder/src/coder.rs`) detects the held key, emits
`MagicZoomCommand` on state transitions, and retargets the camera center
each frame while held. Render layer wraps the app body in an
`egui::LayerId` with `ctx.set_transform_layer(id, TSTransform)`.

### Tests

16 tests under `cargo test -p codelord-core magic_zoom`. No egui harness
needed — pure state machine with mockable delta time. Coverage:

- `Eased` primitive: initial value, terminal value, midpoint symmetry,
  retarget idempotency, retarget-from-current-not-start.
- `Smoothed` primitive: retarget, asymptotic convergence, bounded lag
  under sustained motion.
- `MagicZoomState`: default idle, engage/disengage round-trip, cursor
  tracking, cursor-stop settle, `is_animating` correctness.
- `update_magic_zoom_system`: message drain, tick path, disengage.

### Known limitations

- Text may blur during motion. egui rasterizes glyphs into a texture
  atlas; `set_transform_layer` scales those textures rather than
  re-rasterizing at a higher resolution. Acceptable for v1 — the blur
  during motion is part of the Screen Studio aesthetic. Crisp-on-settle
  is a candidate for a follow-up.
- No edge clamping. The camera can zoom into empty frame edges near the
  window border. Fix on demand.
